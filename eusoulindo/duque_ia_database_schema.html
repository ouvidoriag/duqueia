Cabeçalhos das Colunas
CID Column ID — número sequencial da coluna dentro da tabela (começa do 0). É a posição da coluna na ordem de criação da tabela.
Coluna O nome do campo conforme definido no banco de dados. É pelo nome da coluna que os dados são acessados nas consultas SQL.
Tipo O tipo de dado esperado nessa coluna (ex: texto, número, data). Define como o dado é armazenado fisicamente e como pode ser consultado.
Not Null Sim = esse campo é obrigatório — o banco rejeita inserções sem valor. Não = o campo é opcional e pode ficar em branco (NULL).
Padrão Valor automático preenchido pelo banco caso nenhum valor seja informado na inserção. Ex: CURRENT_TIMESTAMP grava automaticamente a data/hora atual.
Chave PK = Chave Primária (Primary Key) — identifica cada linha de forma única e não pode se repetir. Toda tabela deve ter uma PK.
Descrição do Campo Explicação funcional do campo — o que o dado representa no contexto do sistema Duque IA e como ele é utilizado.
Tipos de Dados (SQLite)
TEXT Texto livre de qualquer tamanho. Usado para nomes, descrições, endereços, embeddings em JSON, respostas da IA, etc.
INTEGER Número inteiro sem casas decimais (ex: 1, 42, 1000). Usado para IDs, contadores, flags booleanas (0 = Não, 1 = Sim).
REAL Número decimal com ponto flutuante (ex: 0.92, 145.7). Usado para scores de confiança, latência em milissegundos, pesos de relevância.
TIMESTAMP Data e hora combinadas (ex: 2026-06-30 14:32:00). Registra quando algo aconteceu — criação de sessão, envio de mensagem, etc.
BLOB Dados binários brutos (imagens, arquivos). Raramente utilizado diretamente — no Duque IA os arquivos são referenciados por caminho (TEXT), não armazenados como BLOB.
BOOLEAN O SQLite não possui tipo booleano nativo — valores booleanos são armazenados como INTEGER (0 = Falso, 1 = Verdadeiro).
